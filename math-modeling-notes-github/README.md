## AI 整理说明

本仓库在整理为 GitHub 可维护版本的过程中，使用了 AI 工具进行辅助处理；但 AI 的作用主要是**工程化整理与兼容性修正**，而不是对原始笔记内容进行实质性改写。

做了如下两种整理

### 1. 仓库化整理

把原始笔记整理成适合放在 GitHub 仓库中维护的版本。

> 1) 图片引用是 本地绝对路径 / Typora 路径 / HTML img ... 这样散落的 markdown 图片  
> 2) 混用了 markdown、HTML、LaTeX 
> 3) 早期不会文内 LaTeX，很多上下标和数学表达是用 `<sub><sup>` 或纯文本硬凑的  
> 4) 有些图片原本只是忘了换行，但却会保留“文字后紧接图片”的坏排版

- 清理 Markdown / HTML / LaTeX 混写遗留问题

- 将本地图片引用改为仓库内相对路径
- 整理资源目录，恢复并统一图片管理方式
- 调整版式结构，使正文、图片、标题、引用等更适合长期维护
- 不改动原文信息与措辞，但完成源码清理与仓库化重构

### 2. GitHub 渲染兼容修正

专门解决 **GitHub 页面显示问题**。 

> 1. 凡是`$...$`这种内联公式前后没有空格就显示不出来 
>2. 凡是`$$...$$`这种独立公式前后没有空行就显示不出来 
> 
>3. `\bold` 等很多非标准 Latex 语法 无法识别 

- 修正内联公式在 GitHub 中的显示兼容性
- 修正独立公式前后空行、块级公式渲染方式等问题
- 替换不稳定或非标准的数学宏写法
- 调整部分分段函数、矩阵与块公式写法，使其更适合 GitHub 数学渲染环境

### 说明

- AI 参与整理的目标始终是：**尽量不改变原文信息与原有措辞，只处理维护性、显示兼容性与源码规范问题。**
- 若仍存在公式渲染、链接跳转、资源路径或排版细节问题，欢迎继续 `issue` 指出并修正。
- 涉及数学定义、统计结论、软件语法与建模方法时，仍应以课程材料、软件文档与实际使用场景为准。
